Руководство пользователя

Консольная программа для работы с Telegram аккаунтами: рассылка сообщений, рассылка по группам, сбор аудитории из групп и прогрев аккаунтов. Всё управление идёт через простое консольное меню.

1. Запуск программы

Программа запускается двойным кликом или командой:

TGWRAPPER.exe

2. Первый запуск и активация

При первом запуске программа покажет, что лицензия не активна, и выведет HWID привязанный к этому компьютеру.

Что делать:

  1. Скопировать показанный HWID.
  2. Отправить его тому, у кого вы получили программу.
  3. В ответ придёт файл license.key.
  4. Положить этот файл рядом с программой, в ту же папку.
  5. Запустить программу заново.

После этого программа откроет обычное меню и будет работать.

3. Главное меню

После активации появляется меню:

[1] Sender          рассылка в личные сообщения и чаты
[2] Group Sender    рассылка по группам
[3] Parser          сбор участников группы
[4] Warmer          прогрев аккаунтов

[5] Добавить аккаунт
[6] Удалить аккаунт
[7] Показать аккаунты
[8] Сменить прокси
[0] Выход

Чтобы выбрать пункт, нужно ввести его номер и нажать Enter.

4. Аккаунты

Все аккаунты, которые вы добавили, хранятся в одном общем списке.

Добавление аккаунта (пункт 5)

Сначала программа спросит имя для аккаунта, это просто метка, под которой он будет отображаться в списке.

Затем спросит прокси. Прокси можно не указывать, если он не нужен. Поддерживаются такие форматы(Если не указывать тип прокси, то автоматически ставится socks5):

ip:port
ip:port:логин:пароль 
логин:пароль:ip:port
socks5://ip:port
http://ip:port
http://логин:пароль@ip:port

Дальше есть два способа добавить сам аккаунт.

Способ 1. Вход по телефону.
Нужно ввести API ID и API HASH (их можно получить на сайте my.telegram.org, зарегистрировав приложение), затем номер телефона. После этого программа сразу попросит код входа из Telegram, а если у аккаунта есть 2fa, то ещё и пароль нужно будет ввести.

Способ 2. Импорт из tdata.
Если у вас есть папка tdata от Telegram Desktop, можно указать путь к ней. Программа сама превратит её в рабочий аккаунт, вводить код и пароль не нужно, так как аккаунт уже авторизован. Passcode просто проскипайте.

Показать аккаунты (пункт 7)

Выводит таблицу со всеми аккаунтами: имя, API ID, телефон, прокси и статус.

Статус может быть:

Рабочийаккаунт авторизован и без ограничений
Спамблоку аккаунта есть ограничения на рассылку
Отлетелсессия аккаунта больше не работает или прокси не рабочий
Неизвестностатус ещё не проверялся

После показа таблицы программа спросит, нужно ли обновить статусы прямо сейчас. Если согласиться, можно выбрать один аккаунт, несколько (например 1,3-5) или все сразу (all). ВАЖНО! Вводить нужно не имя которым вы назвали аккаунт, номер в таблице (#)

Удалить аккаунт (пункт 6)

Показывает список аккаунтов, нужно ввести номер аккаунта для удаления и подтвердить.

Сменить прокси (пункт 8)

Показывает список аккаунтов, нужно выбрать номер аккаунта, после этого можно ввести новый прокси или оставить поле пустым, тогда прокси у аккаунта будет убран.

Выбор аккаунтов в модулях

Во всех четырёх модулях перед началом работы программа покажет список аккаунтов и попросит выбрать, какими пользоваться. Можно указать:

  • один номер, например 2
  • несколько через запятую, например 1,3,5
  • диапазон, например 1,3-5
  • all, чтобы взять все аккаунты сразу

5. Sender: рассылка в личные сообщения и чаты

Пункт меню 1. Отправляет сообщение по списку юзеров из файла.

Порядок работы:

  1. Выбрать один или несколько аккаунтов.
  2. Указать путь к текстовому файлу с юзерами (по умолчанию программа предложит файл targets.txt из папки input). В файле каждый юзер с новой строки: @username или ID пользователя.
  3. Указать путь к текстовому файлу с текстом сообщения (по умолчанию message.txt). В тексте можно использовать разметку Telegram и спинтакс, подробнее в разделе 10.
  4. По желанию прикрепить фото или файл.
  5. Указать задержку между отправками в секундах (по умолчанию 30).

Список юзеров делится между выбранными аккаунтами поровну. Например если юзеров 10 и выбрано 2 аккаунта, каждый отправит по 5.

Если аккаунт получает спамблок, то юзеры которым он не отправил сообщение раскидаються между другими рабочими аккаунтами, если некуда раскидывать то они добавляються в отдельный файл results/not_sent_<время>.txt.

В конце показывается таблица: сколько отправлено, сколько ошибок, и статус по каждому аккаунту.

6. Group Sender: рассылка по группам

Пункт меню 2. Работает так же, как обычная рассылка, но по списку групп.

Порядок работы:

  1. Выбрать один или несколько аккаунтов.
  2. Указать путь к файлу со списком групп (по умолчанию groups.txt). Каждая строка это @username публичной группы или инвайт-ссылка .
  3. Указать путь к файлу с текстом сообщения, так же как в Sender.
  4. По желанию прикрепить фото или файл.
  5. Указать задержку между отправками.

Если аккаунт не в группе, он сначала вступает в неё сам, публичные группы через обычное вступление, приватные по инвайт-ссылке, и только потом отправляет сообщение. ВАЖНО! У телеграм есть лимиты на вступления, поэтому ставьте больше времени между отправками/p>

Список групп делится между аккаунтами так же как и в лс. Ошибки показываются по каждой группе отдельно: например бан в группе, запрет на запись, истёкшая инвайт-ссылка или ограничение на рассылку. При ограничении на рассылку аккаунт останавливается.

В конце показывается таблица с итогами по каждому аккаунту.

7. Parser: сбор участников группы

Пункт меню 3. Собирает список пользователей группы в текстовый файл.

Есть два режима:

Открытая группа. Список участников виден напрямую, программа просто считывает всех участников группы.

Скрытая группа. Список участников закрыт, тогда программа просматривает последние сообщения в чате и собирает пользователей, которые писали сообщения. Программа спросит, сколько последних сообщений просмотреть (по умолчанию 3000).

Порядок работы:

  1. Выбрать режим (открытая или скрытая группа).
  2. Выбрать один аккаунт.
  3. Указать ссылку на группу или её @username. Если нужно сразу много групп прасить то указать путь к txt файлу с группами
  4. Для закрытых групп указать, сколько сообщений просматривать.

Собранные пользователи сохраняются без повторов. Результат записывается в текстовый файл, названный по имени группы, в двух колонках: ID пользователя и его @username, если он есть.

Если в группе найдено меньше 10 пользователей, то результат не сохраняется, программа выдаст ошибку.

8. Warmer: прогрев аккаунтов

Пункт меню 4. Делает аккаунты более живыми перед началом активной работы. Состоит из двух частей.

Вход в каналы и чаты. Программа заходит в каналы и чаты из подготовленного текстового файла (по умолчанию warmup_chats.txt). Строки такие же, как в списке групп: @username или инвайт-ссылка.

Переписка между своими аккаунтами. Аккаунты, которые вы добавили в программу, пишут друг другу короткие фразы на английском языке. Отдельный файл с фразами не нужен, они уже встроены в программу. Для переписки нужно минимум 2 добавленных аккаунта.

Порядок работы:

  1. Выбрать один или несколько аккаунтов.
  2. Ответить, нужно ли заходить в чаты из файла, и если да, указать путь к файлу.
  3. Указать, сколько времени должен длиться прогрев, максимум 24 часа.

В конце показывается таблица: сколько чатов пройдено и сколько сообщений отправлено по каждому аккаунту (за 24 часа аккаунт отпраляет до 20 сообщений).

9. Файлы, с которыми работает программа

Программа хранит и использует несколько папок и файлов рядом с собой:

accountsздесь хранятся добавленные аккаунты и их сессии, можно перекинуть файлы если хотите быстро добавить сессии на другом устройстве
resultsсюда сохраняются файлы, собранные парсером
inputпапка по умолчанию с данными для рассылки, парсинга и т.д

10. Спинтакс и разметка текста

В текстах для рассылки и прогрева можно использовать спинтакс. Это фигурные скобки с вариантами через вертикальную черту:

{Привет|Здравствуйте|Хай}

Программа каждый раз выбирает один случайный вариант, поэтому сообщения получаются разными.

В текстах для Sender и Group Sender также работает разметка Telegram:

<b>жирный текст</b>
<i>курсив</i>
<a href="https://t.me/имя">ссылка</a>
